HTML Grundlagen

HTML (HyperText Markup Language) entstand Anfang der 1990er Jahre am CERN. Tim Berners‑Lee definierte eine einfache, SGML-inspirierte Auszeichnungssprache, um Dokumente im neu entstehenden World Wide Web zu verlinken. Mit frühen Spezifikationen wie HTML 2.0 (1995) und HTML 4.01 (1999) verbreitete sich das Web rasant und legte die Basis für offene, vernetzte Inhalte.

Heute steht HTML5 für modernes, semantisches HTML. Elemente wie header, nav, main, section, article, aside und footer beschreiben die Bedeutung von Inhalten statt nur deren Aussehen. Dazu kommen integrierte Medien (audio, video), Grafik (canvas, svg), bessere Formulare und APIs. Semantik verbessert Zugänglichkeit, SEO und Wiederverwendbarkeit von Inhalten.

Als Grundgerüst jeder Webanwendung strukturiert HTML die Inhalte, CSS gestaltet das Erscheinungsbild und JavaScript sorgt für Interaktivität. Ob statische Seiten, SPAs mit Frameworks oder serverseitig gerenderte Apps – alles baut auf sauberem HTML auf. Gute Struktur, progressive Enhancement und barrierefreie Markup-Praktiken machen Anwendungen robust und zukunftssicher.

HTML Elemente

Unterschied zwischen Tags und Elementen: Ein Tag ist das Markierungskennzeichen in spitzen Klammern (z. B. <p>), ein Element besteht aus Start‑Tag, optionalem Inhalt und End‑Tag (z. B. <p>Text</p>). Manche Elemente sind leer (void) und haben keinen Inhalt sowie keinen End‑Tag, z. B. <img> oder <br>.

Wichtige HTML‑Elemente und kurze Erklärung
Element Erklärung
html Wurzelelement eines HTML‑Dokuments.
head Metadaten, Titel und Verweise auf Ressourcen.
title Seitentitel in der Browser‑Registerkarte.
meta Metainformationen (z. B. Charset, Viewport, Description).
link Verknüpft externe Ressourcen wie CSS.
script Bindet JavaScript ein.
body Beinhaltet alle sichtbaren Inhalte der Seite.
header, nav, main, footer Semantische Bereiche der Seitengestaltung.
section, article, aside Semantische Inhaltsblöcke und Randinhalte.
h1–h6 Überschriften‑Hierarchie.
p Absatztext.
a Hyperlink zu anderen Seiten oder Bereichen.
img Bild (void‑Element; kein End‑Tag, kein Inhalt).
ul, ol, li Listen: ungeordnet, geordnet und Listeneintrag.
table, thead, tbody, tr, th, td Tabellen und ihre Struktur.
form, label, input, button Formulareingaben und Interaktion.
figure, figcaption Abbildung mit begleitender Beschriftung.
div, span Generische Container (Block bzw. Inline).

Attribute & Regeln: Attribute ergänzen Elemente um Zusatzinformationen (z. B. href, src, alt, class). Sie stehen im Start‑Tag und folgen der Form name="wert". Elemente werden korrekt verschachtelt und geschlossen: die meisten haben ein Start‑Tag und ein End‑Tag; sogenannte void elements (z. B. img, br, meta, link, input) besitzen keinen End‑Tag und dürfen keinen Inhalt haben. Wohlgeformtes Markup (saubere Verschachtelung, passende End‑Tags) verbessert Zugänglichkeit, Wartbarkeit und Browserkompatibilität.

CSS

CSS (Cascading Style Sheets) beschreibt das Aussehen von HTML‑Inhalten: Farben, Schriften, Abstände, Layouts und Animationen. Zusammen mit semantischem HTML und JavaScript ermöglicht CSS responsive, zugängliche und performante Weboberflächen.

Fonts & Farben

Typografie, Webfonts, Farbmodelle (HEX, RGB, HSL) und Kontrast für gute Lesbarkeit.

Boxmodell

Inhalt, Padding, Border, Margin und box-sizing bestimmen Größe und Abstände.

Positioning

Static, relative, absolute, fixed, sticky und z-index für Ebenensteuerung.

Selektoren

Typ‑, Klassen‑, ID‑, Attribut‑, Pseudoklassen/-elemente und Spezifität.

Flexbox

Einfaches 1D‑Layout: Ausrichtung, Verteilung, Reihenfolge und flexible Größen.

Grid

Leistungsstarkes 2D‑Layout: Tracks, Areas und responsive Raster.

Transitions

Weiche Übergänge, Transformationen, Keyframe‑Animationen und Performance‑Tipps.

Übersicht: Bootstrap, Tailwind & Co. – Utility vs. Komponenten‑Ansatz.

Visual Studio Code

Visual Studio Code (VS Code) ist eine schnelle, kostenlose und plattformübergreifende Entwicklungsumgebung, und derzeit der Industriestandard für Webentwicklung.

W3C Validator: Validiert HTML direkt in VS Code nach W3C‑Standards, weist auf Fehler und Barrierefreiheitsprobleme hin.

Live Server: Startet einen lokalen Entwicklungsserver mit automatischem Reload. Änderungen an HTML/CSS/JS sind sofort im Browser sichtbar.

JavaScript

JavaScript ist die Programmiersprache des Webs und sorgt für Interaktivität, dynamische Inhalte und Logik in der Benutzeroberfläche. Es steuert den DOM, reagiert auf Events (Klick, Eingabe, Scroll), kommuniziert über Netzwerke (z. B. fetch) mit Servern und ermöglicht komplexe Anwendungen im Browser. Neben dem Frontend wird JavaScript mit Node.js auch serverseitig eingesetzt.

Die Sprache wird durch ECMA International als ECMAScript standardisiert. Seit ES6/ES2015 kamen zentrale Features hinzu: let/const, Arrow‑Funktionen, Klassen, Module, Promises sowie async/await. Moderne Toolchains (Bundler, Transpiler) und optional TypeScript verbessern Struktur, Wartbarkeit und Entwicklererlebnis.
